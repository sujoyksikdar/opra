Mock Election

The Mock Election tab lets you run ranked-choice voting simulations across congressional districts. You can create polls for specific districts, collect ranked preferences from voters, and view results both per-poll and aggregated across an entire state and year.

Creating a Mock Election Poll

Click "Mock Election" in the sidebar, then click "Create New".

Step 1 — Basic Info
  • 1Question Title — A unique name (e.g. "Texas District 5 – 2026 Election").
  • 2Description — Optional context for voters.
  • 3State — Select the US state this poll belongs to.
  • 4District — Select the congressional district. The dropdown is populated automatically based on the selected state.
  • 5Year — Enter the election year (e.g. 2026). This groups polls together for aggregated results.
  • 6Allow Ties — Whether voters can rank two candidates equally.
Step 2 — Add Candidates
  • 1Enter a candidate name and click "+" to add them. Optionally add a photo.
  • 2Repeat until all candidates are added.
  • 3Click "×" to remove a candidate.
ⓘ All polls in the same state and year should use identical candidate names. Spelling differences (e.g. "Alice Smith" vs "Alice") will be treated as separate candidates in the aggregated results.
Step 3 — Invite Voters
  • 1Search for registered OPRA users by username and add them.
  • 2Add or remove entire groups of users at once.
  • 3Check "Email announcement" to notify users when added.
  • 4Bulk-invite via comma-separated email addresses in the CSV field.
Step 4 — Settings

Configure algorithms, visibility, and UI preferences the same way as regular polls.

Voting in a Mock Election

Voters rank the candidates in order of preference using any available UI (two-column drag, one-column, sliders, stars, etc.). Rank 1 is the most preferred candidate.

Poll Settings
  • Start / Pause / Stop the poll using the status buttons.
  • Recalculate Result — Re-run the voting algorithms after the poll ends.
  • Location — Shows the district, state, and year assigned to this poll.
  • Aggregated Results — Visible once the poll has a state and year set and has been started or ended. Takes you to the statewide aggregated view.
Aggregated Results

The Aggregated Results page combines votes from all polls in the same state and year to show a unified statewide picture. Navigate here from Poll Settings → "Aggregated Results".

District Map
  • An interactive map of the selected state with congressional districts colored by the leading candidate.
  • Algorithm dropdown (top right) — Select any of the 16 supported voting algorithms. The map recolors automatically without a page reload.
  • Algorithms marked (may be slow) are elimination-round algorithms (STV, Baldwin, Coombs, etc.) that take longer to compute.
  • Sidebar (left) — Lists all districts with checkboxes. Uncheck a district to hide it from the map.
  • Hover over a district — Shows the winner and vote breakdown for that district.
  • Grey districts have no votes yet.

Statewide Algorithm Comparison Table

Shows results for all 16 algorithms side by side from combined votes across all polls in the state and year. Useful for comparing how different voting methods affect the outcome.

District Winners Table

Lists each district, its winning candidate, and the first-choice vote breakdown per candidate. Updates when you change the algorithm dropdown.

Filtering and Sorting Polls

On the Mock Election polls list page you can:

  • Search polls by name using the search box.
  • Filter by status using the status pills (All / Active / Draft / Ended / Paused).
  • Filter by Year — Select a year from the Year dropdown to show only polls from that year.
  • Filter by State — Select a state from the State dropdown to show only polls in that state.
  • Sort — Sort polls by newest, oldest, or by election year (ascending or descending).
ⓘ The Year and State dropdowns only appear if you have polls with those fields set.
